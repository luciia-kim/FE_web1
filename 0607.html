web1-16. 원시웹

web 과 internet은 다른 것. web은 internet안에 존재하는 서비스.
1960년 internet의 등장, 30년 후 1990년 web이 등장함.

  • Internet 등장(1960)
    핵공격에도 강인한 통신 시스템의 필요 - 하나의 메인으로 이루어진 것이 아닌, 분산된 형태의 통신시스템이기 때문에 핵공격에도 안전했음. Internet은 web 출현 이전까지는 정부, 군대, 대학 등 큰 조직에서만 사용하는 것이었음.
  • web의 등장(1990)
    swiss 제네바 물리학 연구소 cern의 버너스리 연구원팀이 만듦. 팀 버너스리가 1990년 10월 최초의 웹페이지 편집기 제작, 11월 World Wide Web제작, 12월 24일 info.cern.ch 라는 주소의 최초의 웹페이지를 만듦.



web1-17. 서버와 클라이언트

Web Browser, Web Server 두 대의 컴퓨터.
웹서버 컴퓨터에는 http//info.cern.ch 라는 주소를 갖고 있으며, 하드디스크가 있고, index.html이라는 파일을 갖고 있음.
웹 브라우저 컴퓨터에서는 http://info.cern.ch/index.html 이라는 주소를 입력했을 때, 웹서버 컴퓨터는 index.html 파일의 전기적 신호를 보내주게 됨.
Web Browser 컴퓨터는 Web Server 컴퓨터에 정보를 요청(request)하고, Web Server 컴퓨터는 Web Browser 컴퓨터에 응답(response)한다.
이 관계를 서버와 클라이언트로 정의하는데, 요청하는 컴퓨터가 client, 응답하는 컴퓨터를 server 컴퓨터로 부른다.

Web Browser = (Web/Game/Chatting...) Client / Web Server = (Web/Game/Chatting...) Server

#19-2. 웹서버와 http

bitnami 페이지 주소 localhost 를 http://127.0.0.1/index.html 로 바꿔 입력하면, 같은 페이지가 나옴
bitnami 폴더내에 apache-htdocs 폴더를 들어가면 index.html 파일이 있음. 이것을 메모장으로 열면 스크립트가 보이는데, body 태그 안에 뭔가 적고 리로드하면 해당 문구가 들어간 페이지가 나타남.
http://127.0.0.1/index.html 이 주소에서, 127.0.0.1 는 web browser가 설치된 각 컴퓨터의 웹서버의 IP주소(Internet Protocol Address)
(127.0.0.1 은 전세계 공통으로 약속된 주소임)

htdocs 내의 파일을 삭제하고, 우리가 만든 html 파일을 복사해 붙여넣기하면 apache 웹서버를 사용할 수 있는 것

보이는 페이지는 같은데, 무슨 차이인가?
http://로 시작하는 주소를 입력해서 페이지를 연 경우: 웹 브라우저가 같은 컴퓨터 내의 웹 서버에게 요청해서 페이지를 여는 것.
file://로 페이지를 여는 경우: 웹브라우저가 갖고 있는 파일을 직접 여는 것.(웹서버 개입X)

HTTP(Hyper Text Transfer Protocol)는 웹서버와 웹브라우저가 통신할 때 사용하는 통신규약.
서로다른 컴퓨터의 통신을 위해서는 http가 반드시 필요함.


#19-3. 웹서버와 웹브라우저의 통신

웹브라우저가 웹서버에 요청을 할 때는 IP주소가 필요함.
내 웹서버의 IP주소 찾기: 윈도우의 '네트워크 상태'메뉴에서 확인 가능. wifi의 경우, 연결된 wifi 또는 이더넷에서 속성을 선택하면 IPv4주소가 적힌 페이지가 열림.
*내 컴퓨터 IP: 192.168.0.10

127.0.0.1은 '컴퓨터 자기 자신'을 나타내는 것이기 때문에 내 컴퓨터 IP주소로 검색했을 때와 같은 페이지가 나타남.
두 대의 컴퓨터에서 같은 페이지를 보기 위해서는, 웹브라우저가 설치된 컴퓨터와 웹서버가 설치된 컴퓨터를 같은 네트워크에 접속해야 함.
